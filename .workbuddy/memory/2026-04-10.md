## 甜品店游戏 Bug修复

### 修复三个Bug：
1. **成就弹窗进度条仍在走** → `startCustomerTimer` 的 setInterval 添加 `if (gameData.paused) return;`
2. **没有领取奖励按钮** → 上版本误设 `claimBtn.style.display = 'none'`，改为 `inline-block`；奖励金币改为点击按钮时才发放（`pendingAchievement` 机制）
3. **顾客上菜后闪现** → 重构 `serveCustomer`：上菜后**立即**从数据和DOM移除顾客（不保留"已服务"中间状态），消除闪现

### 新增字段
- `gameData.pendingAchievement`：保存待领取成就引用
- `unlockAchievement` 不再立即发放金币，改为设置 `_pendingReward`
- `claimAchievementReward` 负责发放金币、关闭弹窗、恢复游戏、执行补位

## 成就弹窗UI优化 + 桌面端Tab文字修复

### 成就弹窗
- 奖励金币从"按钮样式"改为"标签展示"：虚线边框+浅色背景+小字"奖励"前缀，明确是信息展示
- 领取按钮改为粉红渐变色，与奖励标签视觉区分清晰

### 桌面版Tab文字问题
- 根本原因：`.tab-nav`/`.tab-content`/`.tab-panel` 原本只在移动端媒体查询里定义，桌面端无样式导致原始文本漏出
- 修复：在全局CSS中 `.tab-nav { display: none }` 桌面端隐藏；`.tab-content { display: contents }` 透传布局；`.tab-panel { display: block }` 桌面全显示
- 移动端媒体查询内恢复所有tab相关样式（display:flex / display:none等）

## 2026-04-10 傍晚：移动端视觉优化 + Bug修复

### 视觉优化
1. **等待区卡片**：改为右上角悬浮卡片（position:absolute），白色半透明背景+阴影
2. **座位计数器**：移入 `.shop-scene` 末尾作为 flex 子元素，不再用 absolute 定位
3. **收银台上移**：bottom 从 0 改为 70px，cupecake 装饰同步上移，避免与顾客重叠
4. **耐心进度条**：从座位内移到座位**上方外部**（top:-14px, left:-4px, right:-4px），扩大宽度避免与订单气泡重叠

### 交互优化
- **移除浮窗 tooltip**：应用户要求去掉长按弹出的信息气泡
- **改为填充式进度条**：座位背景从右往左收缩的渐变层，绿→橙→红，颜色阶段：0-30%绿，30-50%橙，50%+红

### Bug修复
1. **双击上菜 bug**：手机端 touchend 后会触发 mouse 事件导致两次点击 → 在 ontouchend 加 `e.preventDefault()` 阻止
2. **抖动时机调整**：剩余时间 <30%（即 elapsedPercent >= 70）才开始抖动，50-70% 区间填充变红但不抖动
